# Дерево решений с бэггингом на C для управления роботом

В этой версии проекта **консольное меню и весь вывод программы сделаны на английском**, чтобы не было проблем с кодировкой в Windows, VS Code, PowerShell и CMD.

При этом:

- `README.md` написан на русском;
- комментарии в исходном коде написаны на русском;
- сама программа в консоли выводит английский текст;
- названия классов команд робота тоже английские: `STOP`, `TURN_LEFT`, `TURN_RIGHT`, `FORWARD`, `SLOW_DOWN`.

## Что реализовано

Проект реализует дерево решений на языке C для задачи управления роботом по сенсорным данным.

Реализованы следующие требования:

1. **Дерево решений**
   - построение дерева;
   - разбиение по признаку и порогу;
   - обход дерева от корня к листу для получения предсказания.

2. **Классификация**
   - модель предсказывает команду роботу;
   - доступны два критерия разбиения:
     - `Gini`;
     - `Information Gain / Entropy`.

3. **Регрессия**
   - модель предсказывает рекомендуемую безопасную скорость робота;
   - для разбиений используется уменьшение дисперсии, связанное с MSE.

4. **Бэггинг**
   - строится несколько деревьев;
   - каждое дерево обучается на bootstrap-выборке, то есть на случайной выборке с возвращением;
   - для классификации используется голосование деревьев;
   - для регрессии используется среднее значение предсказаний деревьев.

5. **Случайная подвыборка признаков**
   - в каждом узле рассматривается не весь набор признаков, а случайное подмножество;
   - это делает деревья в ансамбле более разными и снижает переобучение.

6. **Визуализация дерева**
   - первое дерево из ансамбля можно сохранить в `tree_classification.dot` / `tree_regression.dot`;
   - если установлен Graphviz, программа автоматически создаёт PNG-картинку;
   - на внутренних узлах показаны признаки и пороги, а в листьях — класс или числовое значение регрессии.

## Структура проекта

```text
src/
  main.c             главное меню, обучение моделей и ввод сенсоров
  decision_tree.c    построение дерева, Gini, Information Gain, MSE, bagging
  decision_tree.h    структуры данных и объявления функций
  dataset.c              генерация датасетов для классификации и регрессии
  dataset.h              объявления функций датасета
  tree_visualization.c   экспорт дерева в DOT/PNG через Graphviz
  tree_visualization.h   объявления функций визуализации

data/
  robot_classification.csv  пример датасета для классификации
  robot_regression.csv      пример датасета для регрессии

Makefile             сборка через make
run_windows.bat      сборка и запуск на Windows
.vscode/tasks.json   задача сборки для VS Code
```

## Как запустить в VS Code на Windows

1. Распакуйте архив.
2. Откройте в VS Code именно папку проекта:

```text
File → Open Folder → decision_tree_c_project_with_visualization
```

3. Откройте терминал:

```text
Terminal → New Terminal
```

4. Запустите файл:

```bat
.\run_windows.bat
```

Важно: не запускайте проект через кнопку `Run Code`, потому что проект состоит из нескольких `.c`-файлов. Надёжнее запускать через терминал.

## Если GCC не найден

Если появляется ошибка, что `gcc` не является внутренней или внешней командой, значит компилятор не установлен или не добавлен в `PATH`.

Нужно установить MinGW-w64 или MSYS2, а затем проверить команду:

```bat
gcc --version
```

Если версия GCC выводится, проект можно запускать.

## Ручная сборка без bat-файла

В терминале из папки проекта можно выполнить:

```bat
gcc -std=c11 -Wall -Wextra -O2 src/main.c src/decision_tree.c src/dataset.c src/tree_visualization.c -lm -o decision_tree_robot.exe
```

Затем:

```bat
.\decision_tree_robot.exe
```

## Запуск на Linux или macOS

```bash
make
./decision_tree_robot
```

## Как работать с программой

После запуска появится меню:

```text
=============================================
 DECISION TREE WITH BAGGING FOR ROBOT
=============================================
1. Train classification using Gini
2. Train classification using Information Gain
3. Train safe-speed regression
4. Enter sensor values and predict
5. Show first dataset rows
6. Export classification tree visualization
7. Export regression tree visualization
0. Exit
Select option:
```

### Пункт 1

Обучает классификационную модель с критерием `Gini`.

Эту часть можно объяснить так:

> Gini измеряет неоднородность классов в узле. Алгоритм выбирает такое разбиение, при котором после деления данные становятся более однородными.

### Пункт 2

Обучает классификационную модель с критерием `Information Gain / Entropy`.

Эту часть можно объяснить так:

> Information Gain показывает, насколько сильно уменьшается энтропия после разбиения. Чем больше прирост информации, тем лучше выбранный признак и порог.

### Пункт 3

Обучает регрессионную модель для предсказания безопасной скорости.

Эту часть можно объяснить так:

> В регрессии дерево предсказывает не класс, а число. В листе хранится среднее значение целевой переменной, а при выборе разбиения используется уменьшение дисперсии/MSE.

### Пункт 4

Позволяет ввести сенсорные данные робота и получить прогноз.

Пример входных данных:

```text
Distance to obstacle, cm: 20
Current speed, m/s: 0.7
Obstacle angle, degrees [-90; 90]: 30
Battery charge, percent: 80
```

Пример результата:

```text
Model prediction:
Robot command: TURN_RIGHT
Recommended safe speed: 0.521 m/s
```

### Пункт 5

Показывает первые строки датасетов для классификации и регрессии.

### Пункт 6

Создаёт визуализацию первого дерева классификационного ансамбля.

После выбора пункта появляются файлы:

```text
tree_classification.dot
tree_classification.png
```

Файл `tree_classification.png` можно открыть прямо в VS Code. Внутренние узлы показывают условие разбиения, например `distance_cm <= 30.5`, а листья показывают итоговый класс: `STOP`, `TURN_LEFT`, `FORWARD` и так далее.

### Пункт 7

Создаёт визуализацию первого дерева регрессионного ансамбля.

После выбора пункта появляются файлы:

```text
tree_regression.dot
tree_regression.png
```

В листьях регрессионного дерева выводится числовое значение `value`, то есть предсказанная безопасная скорость.

## Graphviz для визуализации

Для создания PNG-файлов нужен Graphviz. Проверить установку можно командой:

```bat
dot -V
```

Если команда не найдена, установите Graphviz и добавьте его в `PATH`. После этого перезапустите VS Code.

Если DOT-файл создался, а PNG нет, можно вручную выполнить:

```bat
dot -Tpng tree_classification.dot -o tree_classification.png
```

или для регрессии:

```bat
dot -Tpng tree_regression.dot -o tree_regression.png
```

## Что показать преподавателю

Рекомендуемый порядок демонстрации:

1. Открыть `README.md` и показать список реализованных требований.
2. Открыть `src/decision_tree.c` и показать:
   - функцию расчёта Gini;
   - функцию расчёта энтропии;
   - выбор лучшего разбиения;
   - построение дерева;
   - обучение ансамбля через bagging;
   - голосование деревьев для классификации;
   - усреднение деревьев для регрессии.
3. Запустить программу.
4. Выбрать пункт `1` — обучение по Gini.
5. Выбрать пункт `2` — обучение по Information Gain.
6. Выбрать пункт `3` — обучение регрессии.
7. Выбрать пункт `4` — ввести сенсоры и получить команду робота и безопасную скорость.
8. Выбрать пункт `6` — создать картинку дерева классификации.
9. Выбрать пункт `7` — создать картинку дерева регрессии.

## Краткое объяснение проекта

Проект обучает ансамбль деревьев решений для управления роботом. На вход подаются четыре признака:

- расстояние до препятствия;
- текущая скорость;
- угол препятствия;
- заряд батареи.

Для классификации модель выбирает команду роботу. Для регрессии модель предсказывает рекомендуемую безопасную скорость. Улучшение качества достигается за счёт бэггинга и случайного выбора признаков в каждом узле дерева.
